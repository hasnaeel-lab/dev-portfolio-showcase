

# Portfolio Développeuse — Application de Présentation

## Objectif
Créer un site portfolio professionnel pour une développeuse, servant à présenter ses compétences, projets GitHub, et articles techniques. Le site sera déployable sur un VPS OVH avec un nom de domaine personnalisé.

---

## Page d'accueil (One-page scrollable)

### 1. Section Hero
- Photo/avatar de la développeuse
- Titre et tagline (ex: "Développeuse Web Fullstack | Passionnée IA")
- Liens rapides vers GitHub, LinkedIn
- Bouton CTA vers les projets ou contact

### 2. Section À propos
- Courte bio professionnelle
- Parcours et évolution (timeline visuelle) : de développeuse classique à l'intégration des outils IA
- Compétences techniques sous forme de badges : React, Node.js, Python, Figma, Redux, Git Flow, VS Code

### 3. Section Projets
- Grille de cartes présentant les projets GitHub
- Chaque carte : titre, description courte, technologies utilisées, lien GitHub
- Filtrage par technologie (React, Node.js, Python, etc.)

### 4. Section Blog / Articles
- Liste d'articles techniques avec aperçu (titre, résumé, date)
- Thèmes prévus :
  - Lovable et les outils IA pour développeurs
  - Évolution du métier de développeuse avec l'IA
  - Nouveautés React / Node.js / Python
  - Design System en React
  - Créer une librairie React
  - Redux — bonnes pratiques
  - Git Flow expliqué
  - Fonctionnalités et plugins VS Code
  - Déployer sur GitHub Pages
- Articles codés en dur (statiques) — faciles à mettre à jour dans le code
- Chaque article est une page dédiée avec contenu formaté

### 5. Section Contact
- Formulaire de contact simple (nom, email, message)
- Liens vers LinkedIn, GitHub, email

---

## Design
- Style **moderne et professionnel**, fond clair avec accents colorés
- Responsive (mobile-first) pour une bonne visibilité depuis LinkedIn
- Animations subtiles au scroll
- Navigation sticky en haut avec ancres vers chaque section

## Pages supplémentaires
- Page individuelle pour chaque article de blog
- Page 404 personnalisée

## Pas de backend requis
- Contenu statique (projets et articles codés directement)
- Formulaire de contact via mailto ou service externe
- Pas besoin de base de données ni d'authentification
